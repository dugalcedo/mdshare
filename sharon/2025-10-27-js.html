üéØ The Three Data Types

In JavaScript, we work with three basic types of data:

1. Strings (text)

Strings are text wrapped in quotes. You can use single quotes, double quotes, or backticks:

let teacher = "Doug"
let student = 'Sharon'
let message = `Hello there`
Tip: Use double quotes inside single quotes, or use a backslash: "The cat says \"meow\""

2. Numbers

Numbers don't need quotes. They can be positive, negative, or decimals:

let year = 2025
let temp = -5
let pi = 3.14
let nothing = 0

3. Booleans (true or false)

Booleans are either true or false (no quotes!):

let isRaining = true
let isSunny = false
Important: true is a boolean, but "true" is a string!

üì¶ Variables

Variables are like labeled boxes that store data. There are two ways to create them:

let (can change)

let mood = "happy"
mood = "excited"  // This works!

const (cannot change)

const name = "Sharon"
// name = "Bob"  // This would cause an error!
Tip: Use const when the value shouldn't change, and let when it might change.

üîÑ Reassigning Variables

With let, you can change a variable's value:

Simple reassignment

let score = 10
score = 20  // score is now 20

Changing numbers

There are several ways to increase or decrease a number:

let temp = 10

// All of these add 1:
temp = temp + 1
temp += 1
temp++

// All of these add 5:
temp = temp + 5
temp += 5

// Subtracting works the same way:
temp = temp - 3
temp -= 3
temp--  // subtracts 1
Example: If temp starts at 10, and you do temp += 5, then temp becomes 15.

üé® Naming Variables

Variable names should be clear and use camelCase:

// Good names (camelCase):
let userName = "Sharon"
let totalScore = 100
let isGameOver = false

// Avoid these styles in JavaScript:
let user_name = "Sharon"    // snake_case
let UserName = "Sharon"      // PascalCase
Tip: Start with a lowercase letter, and capitalize the first letter of each new word.

üõ†Ô∏è Methods

Methods are actions you can perform on certain data types. Different types have different methods:

String methods

let name = "sharon"
console.log(name.toUpperCase())  // prints: SHARON

What happens with wrong types?

let age = 25
// age.toUpperCase()  // ERROR! Numbers don't have this method
Remember: Methods only work on certain data types. Strings have string methods, numbers have number methods, etc.

üêõ Reading Errors

When something goes wrong, Node.js tells you with an error message. Here's what to look for:

Example error: 
TypeError: age.toUpperCase is not a function
    at script.js:23:10

What it means:

  • TypeError - The type of error
  • age.toUpperCase is not a function - What went wrong
  • script.js:23 - The file and line number where the error happened
Tip: Always read the line number in the error message. It tells you exactly where to look!
